Welcome to OctoAcme! This checklist helps new team members get up to speed quickly, regardless of their role. Complete each section to ensure you have the access, context, and connections needed to contribute effectively.

## Project Tool Access

Ensure you have access to all necessary tools and systems:

### Core Development Tools
- [ ] GitHub organization and repositories (with appropriate permissions)
- [ ] GitHub Copilot and Copilot Spaces access
- [ ] Development environment setup (IDE, languages, frameworks)
- [ ] CI/CD pipeline access and permissions

### Project Management and Collaboration
- [ ] Project board access (GitHub Projects or equivalent)
- [ ] Team communication channels (Slack, Teams, or equivalent)
- [ ] Calendar access for team meetings and ceremonies
- [ ] Document repository access (where project artifacts are stored)

### Role-Specific Tools
- [ ] **Developers**: Code repository write access, test environment access, local development setup
- [ ] **Product Managers**: Product analytics tools, user research platforms, roadmap tools
- [ ] **Project Managers**: Project planning tools, time tracking systems, reporting dashboards
- [ ] **UX Designers**: Design tools (Figma, Sketch, etc.), prototyping platforms, user testing tools
- [ ] **Data Analysts**: Data warehouse access, BI tools, experiment platforms, analytics dashboards
- [ ] **DevOps Engineers**: Cloud platform access (AWS, Azure, GCP), monitoring tools, infrastructure repositories
- [ ] **Customer Support Liaison**: Support ticketing system, CRM access, knowledge base editor, customer communication tools

### Access Verification
- [ ] Confirm you can view existing projects and artifacts
- [ ] Test that you can create/edit items in your role's primary tools
- [ ] Verify notification settings are configured appropriately

## Communication Expectations

Understand how, when, and where the team communicates:

### Regular Ceremonies and Meetings
- [ ] Added to daily standup meeting (if applicable to your role)
- [ ] Invited to sprint planning sessions
- [ ] Included in sprint review/demo meetings
- [ ] Added to retrospective sessions
- [ ] Invited to relevant weekly sync meetings

### Communication Channels
- [ ] Joined team communication channels (general, role-specific, project-specific)
- [ ] Understand when to use synchronous vs. asynchronous communication
- [ ] Know the escalation path for urgent issues
- [ ] Familiar with team communication hours and response time expectations

### Status Reporting
- [ ] Understand your role's status reporting requirements (what, when, to whom)
- [ ] Know where to document progress (project boards, status documents, etc.)
- [ ] Familiar with risk and blocker reporting procedures
- [ ] Understand decision documentation practices

### Cross-Role Communication
- [ ] Review the [Cross-Functional Collaboration](./octoacme-roles-and-personas.md#cross-functional-collaboration) section
- [ ] Identify which roles you'll interact with most frequently
- [ ] Schedule introductory meetings with key cross-functional partners
- [ ] Understand handoff points between your role and others

## Team Introductions and Onboarding Buddy

Connect with your team and establish support relationships:

- [ ] Meet with your manager to discuss role expectations and goals
- [ ] Assigned an onboarding buddy or mentor
- [ ] Introduction to immediate team members
- [ ] Introduction to key cross-functional partners
- [ ] Schedule 1:1s with relevant stakeholders

---

## First Week Goals

Set yourself up for success in your first week:

- [ ] Complete all documentation review items
- [ ] Verify all tool access and permissions
- [ ] Attend at least one of each recurring team meeting
- [ ] Shadow a team member in your role (if available)
- [ ] Review current active projects and their status
- [ ] Identify a small, low-risk task to complete as your first contribution
- [ ] Ask questions! Document unclear items and raise them with your buddy or manager

---

## First Month Goals

Build deeper understanding and begin contributing meaningfully:

- [ ] Completed at least one meaningful contribution in your role
- [ ] Participated in a full project phase (or cycle) from start to finish
- [ ] Received and incorporated feedback on your work
- [ ] Provided input in cross-functional discussions
- [ ] Identified areas where documentation could be improved (share with team)
- [ ] Feel comfortable navigating project artifacts and tools independently

---

## Questions and Feedback

Use this space to track questions, blockers, or feedback as you onboard:

- **Questions**: Document anything unclear and bring to your 1:1s or team meetings
- **Blockers**: Note any access issues, missing documentation, or gaps in the onboarding process
- **Feedback**: Share suggestions for improving this checklist or the onboarding experience
